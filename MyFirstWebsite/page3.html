Common Examples

To create a repository: create a new directory, open it and open Git Bash. After entering Git Bash terminal window, initialize the repository with:

git init

Clone a repository: git clone /path/to/repository
Clone a remote repository:
git clone username@host:/path/to/repository

Add file: git add "filename" or add all files git add .

Commit files: git commit -m "commit message"

Push committed files to remote repository:
git push origin master
"origin" is the alias name for the remote repository, which is generated by default when cloning a repository. As the name implies, it directs the commit to the repository where the cloned repository came from.

"master" is the name of the main default branch which is the base of all other side-branches that are created later. Committed files go to the branch specified in the command, for example, if the master branch has a branch named "feature", then the commit command would be:
git push origin feature

Add a remote server: git remote add origin "server"

Create a new branch and switch to it: git checkout -b "branch name"
Switch back to master: git checkout master
Merge a branch: git merge "branch name"
Delete a branch: git branch -d "branch name"

Push branch to remote repository: git push origin "branch name"
Fetch commit history from remote repository: git fetch origin
"fetch" command returns the commit history from remote server to the local repository, after this, git status can be used to check if either local or remote repository has more commits than the other. This helps keeping the code up-to-date and also helps avoiding conflicts if someone else has modified or added new files, which could cause problems for your own modifications.

git status shows if there are untracked or modified files, and files that are staged for next commit. Also, it shows which branch the user currently is in and the status between local and remote repositories commit history.

Pull a remote repository: git pull origin master
This command fetches and merges the master branch from origin repository to the current branch in local repository.

Tag a release point: git tag 1.0.0 1b2e1d63ff
This command puts tag "1.0.0" to the commit with "id" of 1b2e1d63ff.
So it would be "version 1.0.0" of the developed project.

Show repository history: git log This command shows the history of commits, the SHA-1 hashed id's of commits and the commit author names along with time and date.

Reset changes of specified file: git checkout -- "filename"
This command resets the specified file's changes. The modified file is removed and replaced by its previous committed version.
